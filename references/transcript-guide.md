# Transcript Guide

Read this file when transcribing audio/video for captions, or when importing existing transcripts.

---

## How Transcripts Are Generated

The primary transcription pipeline uses the video script's built-in TTS, which produces perfectly timed text. For existing audio/video that needs transcription, use external tools.

---

## Built-In TTS Transcription (Recommended)

When using `html2video.py` with `--export-script`, the pipeline generates the narration text automatically with perfect timing. No separate transcription needed.

```bash
python scripts/html2video.py input.html --export-script
```

This produces a text file with the narration script and slide-to-text mapping.

---

## External Transcription

For transcribing existing audio/video that wasn't generated by this skill:

### Option 1: OpenAI Whisper API (Recommended for Quality)

```bash
curl https://api.openai.com/v1/audio/transcriptions \
  -H "Authorization: Bearer $OPENAI_API_KEY" \
  -F file=@audio.mp3 -F model=whisper-1 \
  -F response_format=verbose_json \
  -F "timestamp_granularities[]=word" \
  -o transcript.json
```

Produces word-level timestamps — best for per-word caption animation.

### Option 2: Local Whisper (Offline)

```bash
pip install openai-whisper
whisper audio.mp3 --model small --language zh --output_format json
```

### Option 3: Groq Whisper API (Fast, Free Tier)

```bash
curl https://api.groq.com/openai/v1/audio/transcriptions \
  -H "Authorization: Bearer $GROQ_API_KEY" \
  -F file=@audio.mp3 -F model=whisper-large-v3 \
  -F response_format=verbose_json \
  -F "timestamp_granularities[]=word" \
  -o transcript.json
```

## Supported Input Formats

| Format | Extension | Word-level? |
|---|---|---|
| Whisper JSON | `.json` | Yes |
| SRT subtitles | `.srt` | No (phrase-level) |
| VTT subtitles | `.vtt` | No (phrase-level) |
| OpenAI API JSON | `.json` | Yes |

**Word-level timestamps produce better captions.** SRT/VTT give phrase-level timing, which works but can't do per-word animation effects.

## Whisper Model Guide

| Model | Size | Speed | Accuracy | When to use |
|---|---|---|---|---|
| `tiny` | 75 MB | Fastest | Low | Quick previews |
| `base` | 142 MB | Fast | Fair | Short clips, clear audio |
| `small` | 466 MB | Moderate | Good | **Default** — most content |
| `medium` | 1.5 GB | Slow | Very good | Important content, noisy audio |
| `large-v3` | 3.1 GB | Slowest | Best | Production quality |

### Language Selection

**Critical: `.en` models translate non-English audio into English** — they don't transcribe it. If the audio might not be English, always use a model without the `.en` suffix.

```bash
# Chinese audio
whisper audio.mp3 --model small --language zh

# Unknown language — let whisper auto-detect
whisper audio.mp3 --model small
```

**Music and vocals over instrumentation**: Use `medium` as the minimum. For music videos, providing known lyrics as an SRT/VTT file will always beat automated transcription.

## Transcript Quality Check (Mandatory)

After every transcription, **read the transcript and check for quality issues before proceeding.** Bad transcripts produce nonsensical captions.

### What to look for

| Signal | Example | Cause |
|---|---|---|
| Music note tokens (`♪`) | `{ "text": "♪" }` | Whisper detected music, not speech |
| Garbled / nonsense words | "Do a chin", "huh" | Model misheard lyrics or noise |
| Long gaps with no words | 20+ seconds of only `♪` tokens | Instrumental section |
| Repeated filler | Many "huh", "uh" entries | Model hallucinating on music |
| Very short word spans | `end - start < 0.05` | Unreliable timestamps |

### Automatic retry rules

**If more than 20% of entries are music tokens or obvious nonsense, the transcription failed.** Do not proceed. Instead:

1. **Retry with `medium`** if the original used `small` or smaller
2. **If `medium` also fails** — the audio is too noisy for local transcription. Suggest:
   - Providing text manually as an SRT/VTT file
   - Using an external API (OpenAI or Groq Whisper)
3. **Always clean the transcript** before building captions — filter out music tokens and single non-word characters

### Cleaning a transcript

```js
var raw = JSON.parse(transcriptJson);
var words = raw.filter(function (w) {
  if (!w.text || w.text.trim().length === 0) return false;
  if (/^[♪\u266a\u266b\u266c\u266d\u266e\u266f]+$/.test(w.text)) return false;
  if (/^(huh|uh|um|ah|oh)$/i.test(w.text) && w.end - w.start < 0.1) return false;
  return true;
});
```

## Transcript Format for Captions

The caption system expects this format:

```json
[
  { "text": "Hello", "start": 0.0, "end": 0.5 },
  { "text": "world.", "start": 0.6, "end": 1.2 }
]
```

Convert from other formats as needed. See [captions.md](captions.md) for how to use this in GSAP timelines.

---

## If No Transcript Exists

1. Check the project root for `transcript.json`, `.srt`, or `.vtt` files
2. If none found, and you have audio/video to transcribe:
   - Speech/voiceover → `small` model
   - Music with vocals → `medium` model
   - Chinese content → `small --language zh`
3. **Read the transcript and run the quality check** (see above)
4. If it fails, retry with a larger model or suggest manual text input
